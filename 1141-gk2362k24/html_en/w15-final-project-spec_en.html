Section 2: Game Design Focus

Slide 2.1: The 4 Pillars

Game Design Standards: Required Elements

1. N+1 Variable System:
  • Besides basic A/B options (N), the backend must calculate at least one hidden value (+1) (such as favorability, physical strength, money, morality values, etc.) and affect endings accordingly.
2. Historical Research:
  • Story content must be based on real historical backgrounds (specific dynasties, events, or laws), not completely fictional.
3. Win/Loss Judgment:
  • The game must have clear victory and loss conditions, not just a single ending.
4. Tool Application:
  • Use specified GenAI tools to assist in generating code, text, and artwork.
Speaker Notes: Your games can't just be click options and see stories. The backend must have "math" running. For example, if a player chooses a certain option, on the surface nothing happens, but the backend deducts 10 points from "public support". When public support drops below 0, the game forces an end. This is the N+1 variable system.

Section 4: Game Rules

Slide 4.1: Core Interactive Loop

Game Logic Architecture Diagram

Process Steps:
  1. Situation Setup: Display historical background and current events.
  2. Player Decision: Provide 2-3 options.
  3. Variable Calculation (N+1): Update backend values based on choices (e.g., funds -50, morale +10).
  4. Judgment Check: Check if values trigger endings (e.g., funds <= 0 → bankruptcy ending).
  5. Narrative Progression: Move to next situation or ending screen.
Speaker Notes: This is the core logic you need to write for AI: Display situation → Player chooses → Update values → Check if they lost → Continue to next situation. Make sure your prompts clearly describe this loop.
