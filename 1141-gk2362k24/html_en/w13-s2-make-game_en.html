Q-Learning Core Formula and Concepts

New Information = Old Information + Learning Rate × (Real Reward + Information Gap)
NewQ = OldQ + Learning_Rate × (Reward + NewQ - OldQ)

Translation: "I originally thought this restaurant was only worth 50 points. Today I went and found it was amazing (+10 points), and I heard the table next to us had a hidden dessert (+future expectation). So, I need to revise my rating of this restaurant and bump it up a bit!"

AI's Two Learning Phases

Random Guessing Phase (Exploration)

When AI encounters a problem, it doesn't know the answer, so it rolls the dice and randomly chooses an action.

Result: If it catches the ball (+10 points), it quickly notes on its cheat sheet: "This move works!"

Expert Phase (Exploitation)

After playing thousands of times, the cheat sheet is full. When AI sees a problem, it directly looks up the table and chooses the action with the highest score.

Objective

Prompt Engineering

Prompt Example

Please act as a senior Web game engineer. I want to create a single-file HTML5 game that demonstrates reinforcement learning (RL).

**1. Game Basics:**
* **Type**: Catch the falling object.
* **Controls**: Support "touch swipe/mouse follow" - wherever your finger is, the paddle follows.
* **Rules**: Catching +10 points, missing -10 points.

**2. AI Core (Q-Learning):**
* **State**: Simplified to "horizontal distance interval between Target and Player" (divided into 10 segments). Don't use pixels.
* **Actions**: Move left, move right, stay still.
* **Mechanism**: Implement Q-Learning algorithm with Epsilon-Greedy strategy.

**3. Interface Features (UI):**
* **Toggle Switch**: [Manual Mode] (green) / [AI Mode] (blue).
* **Speed Button**: In AI mode, provide a "Fast Train" button that closes rendering and fast-forwards 2000 rounds.
* **Dashboard**: Display score, round count, exploration rate (Epsilon).

Please provide the complete HTML code.
